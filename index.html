Professional Self-Assessment

My chosen artifact from the IT 145 course is my final project, a comprehensive Java application for an animal tracking company, Grazioso Salvare. This project was a defining moment in my academic journey, solidifying my understanding of computer science principles and sparking my passion for Java.

The project required me to implement a class (Monkey.java) and modify an existing driver class (Driver.java) to manage and track rescue animals. Before this, Java felt like just another programming language, a tool to complete assignments. However, the complexity and scope of the final project transformed my perspective. I had to apply foundational concepts like object-oriented programming (OOP) using inheritance to create the Monkey class, and encapsulation with accessors and mutators, in a real-world, structured context. This experience was the first time I truly appreciated how these abstract concepts work together to build a functional, organized application.

What I came to love about Java was its logical structure and robustness. The clear syntax and the strict nature of a statically-typed language forced me to write cleaner, more predictable code. The process of building on an existing framework, troubleshooting issues, and implementing new features was incredibly rewarding. I learned that coding wasn't just about syntax; it was about problem-solving, creating a logical flow, and building a system that could be understood and maintained by others. This project was more than just a final grade; it was the catalyst that turned a requirement into a passion for software development and cemented my appreciation for the power and elegance of Java.

For my ePortfolio, I will use this single Java artifact to demonstrate a comprehensive skill set across multiple domains. My enhancement plan showcases my abilities in software design and engineering, algorithms and data structures, and databases.

To begin, I will enhance the project for software design and engineering. I will refactor the existing code to improve its modularity and maintainability, ensuring it is a sustainable solution that can be easily understood and managed by other developers. I'll add comprehensive JavaDoc comments to clearly explain the purpose and functionality of each class and method.

For algorithms and data structures, I will optimize the project's performance. Instead of a simple linear traversal, I will enhance the application to use a more efficient binary search algorithm to quickly locate specific animal records. This demonstrates my understanding of algorithmic complexity and my ability to choose the most efficient approach for a given problem.

Finally, for the databases category, I will evolve the project from a simple in-memory application to one that uses a persistent database. My enhancement plan is to implement a SQL database using JDBC to handle data storage and retrieval. This demonstrates my skills in designing database schemas and performing CRUD (Create, Read, Update, Delete) operations, thereby fulfilling the course outcome of creating database solutions for real-world problems.

To complete the user experience, I will implement a user-friendly UI. The current application is command-line based, so I will develop a graphical user interface (GUI) using Java Swing. This UI will allow users to intuitively add, update, and search for animal records without needing to interact with the command line. This enhancement demonstrates my ability to design and implement interactive applications.

Collectively, these enhancements to a single project demonstrate my ability to take a solution from concept to a professional, scalable product. This process involves skills such as communicating with stakeholders by documenting my design decisions for a future team and troubleshooting complex issues. By enhancing this one artifact, I will showcase my full range of abilities in software engineering, algorithms, databases, and user interface design, proving my readiness for a career in software development.
