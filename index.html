Enhancement 2: Algorithms and Data Structures

This enhancement focuses on improving the efficiency and complexity of my project's algorithms and data structures.

Description of Enhanced Artifact:

The artifact I chose to enhance is the Rescue Animal System, originally developed in IT 145 and expanded in later courses. It began as a simple command-line program for intaking, reserving, and viewing animals in training. For this milestone, I focused on the algorithms and data structures within the program by restructuring it to use HashMaps instead of arrays. This change made searching, adding, and removing animals more efficient by allowing direct lookups through unique IDs.

Narrative

Justification for Inclusion
This artifact demonstrates my ability to apply appropriate data structures to real problems. Moving from arrays to HashMaps improved performance and scalability as the system grew. I also refined filtering and searching logic with Java streams to improve readability. These changes highlight my ability to balance efficiency and maintainability while choosing the right tools for the task.

Course Outcomes
This enhancement aligns with my plan to demonstrate algorithmic principles and the use of efficient data structures. Replacing arrays with HashMaps reduced lookup complexity, and streams made filtering easier to read while still effective. The improvements show progress toward the outcome of designing and evaluating computing solutions that apply data structure and algorithm knowledge.

Reflection on the Process
Through this enhancement, I learned how much impact the right data structure has on performance and scalability. Arrays were functional in the original version but inefficient for larger datasets. Using HashMaps allowed faster lookups and cleaner management. A challenge I faced was keeping unique IDs consistent, which I solved by resetting the counter based on the highest existing ID. Overall, the process reinforced the importance of designing with growth in mind and gave me valuable practice applying data structures in a professional way.

Enhancement 3: Databases

This enhancement focuses on improving database integration and ensuring persistence in the Rescue Animal System.

Description of Enhanced Artifact:

For this milestone, I enhanced the Rescue Animal System by integrating it with a SQLite database for persistent storage. Instead of relying only on in-memory data, the system can now save and retrieve animal records across sessions. I also added a DatabaseHelper and DatabaseConnector class to handle queries and connections, ensuring clean separation of concerns.

Narrative

Justification for Inclusion
I selected this enhancement because persistent storage is critical in real-world applications. Adding a database shows my ability to design solutions that maintain data integrity and scale beyond simple in-memory structures. It also demonstrates my ability to write SQL queries and integrate them into Java through JDBC.

Course Outcomes
This enhancement demonstrates my ability to apply database management techniques and create reliable data persistence solutions. It aligns with outcomes related to applying knowledge of databases, ensuring robust CRUD operations, and supporting the long-term use of the application.

Reflection on the Process
I learned how to structure code to keep database logic separate from core business logic, which improves maintainability. One challenge was handling exceptions and keeping the system user-friendly when database errors occurred. I resolved this by adding clear error messages and fallbacks. Overall, this enhancement gave me practical experience with integrating databases into applications, which will be essential for professional development.
